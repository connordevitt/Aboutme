Critical Security Vulnerabilities in SolarWinds and Palo Alto Networks

Addressing the latest security patches and vulnerabilities affecting major software platforms

By Connor Devitt

Introduction

Recent security disclosures have highlighted critical vulnerabilities in both SolarWinds Web Help Desk and Palo Alto Networks' Cortex XSOAR. These flaws could potentially allow attackers to execute arbitrary code, posing significant risks to affected systems. It is crucial for users to understand these vulnerabilities and apply the necessary patches to safeguard their systems.

SolarWinds Web Help Desk Vulnerability (CVE-2024-28986)

SolarWinds has released patches to address a critical security vulnerability in its Web Help Desk software, identified as CVE-2024-28986. This vulnerability, with a CVSS score of 9.8, is a deserialization bug that could be exploited to execute arbitrary code on vulnerable instances.

According to SolarWinds, "SolarWinds Web Help Desk was found to be susceptible to a Java deserialization remote code execution vulnerability that, if exploited, would allow an attacker to run commands on the host machine." Although reported as an unauthenticated vulnerability, SolarWinds has been unable to reproduce it without authentication after thorough testing.

The flaw impacts all versions of SolarWinds Web Help Desk including and prior to 12.8.3, and it has been addressed in hotfix version 12.8.3 HF 1. This vulnerability has also been added to the U.S. Cybersecurity and Infrastructure Security Agency (CISA)'s Known Exploited Vulnerabilities (KEV) catalog, indicating active exploitation. Federal agencies are required to apply the fixes by September 5, 2024.

Palo Alto Networks Cortex XSOAR Vulnerability (CVE-2024-5914)

In addition to the SolarWinds vulnerability, Palo Alto Networks has patched a high-severity vulnerability affecting Cortex XSOAR, tracked as CVE-2024-5914 with a CVSS score of 7.0. This command injection issue could allow an unauthenticated attacker to execute arbitrary commands within the context of an integration container.

The flaw affects all versions of Cortex XSOAR CommonScripts before 1.12.33. According to Palo Alto Networks, "To be exposed, an integration must make use of the ScheduleGenericPolling or GenericPollingScheduledTask scripts from the CommonScripts pack."

Additional Vulnerabilities in Palo Alto Networks

Palo Alto Networks also addressed two moderate-severity issues:

  • CVE-2024-5915 (CVSS score: 5.2) - A privilege escalation vulnerability in the GlobalProtect app on Windows devices that allows a local user to execute programs with elevated privileges.
  • CVE-2024-5916 (CVSS score: 6.0) - An information exposure vulnerability in PAN-OS software that enables a local system administrator to access secrets, passwords, and tokens of external systems.

Users are strongly recommended to update to the latest versions to mitigate potential risks. It is also advised to revoke the secrets, passwords, and tokens configured in PAN-OS firewalls after the upgrade as a precautionary measure.

Conclusion

The recent disclosures of critical vulnerabilities in SolarWinds Web Help Desk and Palo Alto Networks Cortex XSOAR underscore the importance of timely patching and updating of software systems. By staying vigilant and applying these updates, organizations can protect themselves against potential exploits and maintain the security of their IT infrastructure.
